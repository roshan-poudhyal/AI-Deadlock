Banker's Algorithm

What is Banker's Algorithm?

The Banker's Algorithm is a resource allocation and deadlock avoidance algorithm developed by Edsger Dijkstra. It tests for safety by simulating the allocation of predetermined maximum possible amounts of all resources, and then checks if allocation requests can be satisfied with the available resources.

The algorithm gets its name from the banking system where a bank never allocates available cash in such a way that it can no longer satisfy the needs of all its customers.

How it Works

The Banker's Algorithm uses the following data structures:

  • Available: Vector of length m indicating available resources
  • Max: n×m matrix defining maximum demand of each process
  • Allocation: n×m matrix defining resources currently allocated to each process
  • Need: n×m matrix indicating remaining resource needs of each process

Safety Algorithm

  1. Let Work and Finish be vectors of length m and n respectively.
  2. Initialize Work = Available and Finish[i] = false for all i.
  3. Find an i such that both Finish[i] = false and Needi ≤ Work. If no such i exists, go to step 5.
  4. Work = Work + Allocationi and Finish[i] = true. Go to step 3.
  5. If Finish[i] = true for all i, then the system is in a safe state.
